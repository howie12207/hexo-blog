學習Nuxt

發表於 2020-04-27 分類於 nuxt

為什麼要使用 Nuxt?

會使用 Vue 之後，要開始了解如何再 SPA 專案上加上 SSR，而普遍最常看到的就是 Vue+Nuxt 的應用，其最主要就是可以拿來解決 SSR SEO 的問題。

環境建置

全新

建專案

npx create-nuxt-app awesome-nuxt-app

運作

cd <project-name>
npm run dev

原有專案添加

添加 Nuxt

cd <project-name>
npm add nuxt

新增 Nuxt 指令

/* package.json */
{
    "name": "my-vue-spa-project",
    "script": {
        "dev": "nuxt"
    }
}

建立頁面

mkdir pages

運作

npm run dev

遷移

依配置將 components 及 views 搬家

資料夾說明

  • assets: 放需要被編譯的靜態資源
  • static: 放不需要被編譯的靜態資源
  • pages: 對應的頁面元件
  • components: 放個別小組件
  • nuxt.config.js: Nuxt 全域設定檔
  • nuxt: Nuxt 暫存資料夾
  • layout: 渲染畫面的外層
  • middleware:
  • plugins:
  • store:

Nuxt 的 router

對 Vue 來說需要到/router/index.js 內修改對應路徑；而 Nuxt 則是直接在 page 資料夾下根據資料夾配置路徑。
其中若要表動態路由，則在前方加上底線(ex: _id.vue)

Nuxt 的頁面結構

切成三部分(由外層至內層)

  • Document: 文件主體
  • Layout: 給頁面元件套用的布局元件
  • Page: 對應路由的頁面元件
Document 最外層

預設 /.nuxt/views/app.template.html

<!DOCTYPE html>
<html {{ HTML_ATTRS }}>
    <head>
        {{ HEAD }}
    </head>
    <body {{ BODY_ATTRS }}>
        {{ APP }}
    </body>
</html>
Layout (中間層)

預設 /layouts/default.vue

假設登入頁需要配置 header 與 footer 兩組件，則可在/layouts 新增 front.vue 檔，將 header 與 footer 在此載入。

<template>
    <div>
        <Header/>
        <nuxt/>
        <Footer/>
    </div>
</template>

<script>
    import Header from '~/components/Header.vue'
    import Footer from '~/components/Footer.vue'

    export default {
        components: {
            Header,
            Footer
        }
    }
</script>
Page (最內層)

在 login.vue 檔內將 layout 改成 front

login.vue

<template>
    <div>
        ...
    </div>
</template>

<script>
    export default {
        layout: 'front'
    }
</script>

客製 SEO 資訊

全域 (nuxt.config.js)

head: {
  title: '網站名稱',
  meta: [
    { charset: 'utf-8' },
    { name: 'viewport', content: 'width=device-width,     initial-scale=1' },
    { 'http-equiv': 'X-UA-Compatible', content:'ie=edge' },
    { property: 'og:title', content: '標題' },
    { property: 'og:description', content: '敘述' },
    { property: 'og:site_name', content: '網站名稱' },
    { property: 'og:locale', content: 'zh_TW' }
  ]
}

個別畫面(pages/*.vue)

<template>
...
</template>
<script>
    export default {
        head: {
          title: '標題'
        },
        /*...*/
    }
</script>

Nuxt 生命週期

  • nuxtServerInit (Store action)

  • middleware 中間件

    這頁執行哪些 middleware。

  • validate 驗證

    進入頁面前的驗證規則。false 代表沒驗過，Nuxt 會導到錯誤頁面。

  • asyncData

    頁面元件渲染前執行，回傳 Promise 物件，會把 Promise callback 最終回傳值設為頁面 data。

  • fetch

    與 asyncData 同，不同的是回傳值不做處理。

  • Render

驗證錯誤頁面

在一般元件內新增以下代碼，即可導到報錯頁面

<script>
  validate(){
      return false;
  }
</script>

此時頁面預設為 .nuxt/components/nuxt-error.vue

可自己客製頁面，在 layouts/error.vue 新增

<template>
  <div class="container">
    <h1 v-if="error.statusCode === 404">Page not found</h1>
    <h1 v-else>An error occurred</h1>
    <nuxt-link to="/">Home page</nuxt-link>
  </div>
</template>

<script>
export default {
  props: ['error'],
  layout: 'blog' // you can set a custom layout for the error page
}
</script>

客製讀取效果

預設是個進度條 (Progress Bar) 效果，檔案在 .nuxt/components/nuxt-loading.vue

針對原進度條元件調整

nuxt.config.js

loading: {
  color: 'black', //正常進度條顏色
  failedColor: 'red',
  height: '2px',
  duration: 5000,
  rtl: false
},

更換成客製讀取效果

nuxt.config.js

loading: `~/components/loading.vue`,

手動控制各頁面效果

pages/login.vue

<script>
    export default {
        /* ... */
        loading: false
    }
</script>

然後手動呼叫讀取效果觸發

$nuxt.$loading.start(); //顯示進度條
$nuxt.$loading.end(); //結束進度條

客製轉場效果

assets/transition.css

.page-enter-active, .page-leave-active {
  transition: opacity .5s
}
.page-enter, .page-leave-to {
  opacity: 0
}
.page-enter-to, .page-leave {
  opacity: 1
}

nuxt.config.js

export default {
  css: ['~/assets/transition.css']
}
