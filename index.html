設定 SSH

為什麼要使用 SSH ?

之前上傳到 github 都是透過 https 的方式，而這次製作 hexo 專案，要透過 SSH 將檔案上傳到 github 上，所以就需要知道如何使用 SSH。

1. 開啟終端機

ssh-keygen -t rsa -b 4096 -C "howie12207@gmail.com"

出現以下畫面

Generating public/private rsa key pair.
Enter a file in which to save the key (/Users/you/.ssh/id_rsa): [Press enter]
Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]

第二行是 存放 key 檔案的位置
第三行與第三行是 輸入密碼與確認密碼
若沒有特別需要，都按 Enter 直接過去即可

2. 開啟 ssh-agent

這裡奇怪的是我使用 VScode 的 powershell 無法正常操作
後來改成 git bash 就正常了

eval "\$(ssh-agent -s)"

3. 將私鑰加入到 ssh-agent

ssh-add -K ~/.ssh/id_rsa

4. 上傳公鑰

將公鑰的內容打印出來並複製

cat ~/.ssh/id_rsa.pub

到 Github 上 -> setting -> SSH andGPG keys -> New SSH key

將內容貼上，然後自己取名

開始使用 Hexo

為什麼開始使用 Blog?

在這學習程式語言的階段，有太多東西需要去學習，而記憶力就只有那麼一點，透過部落格的紀錄方式，方便自己以後回來查找，也可以記錄一些自己的生活。

部落格是怎麼製作的?

爬了一下文，發現大部分的人都是使用’hexo’，所以就找了幾篇關於 hexo 的教學文來看

建置

1. 開啟 VScode，在終端機中使用以下指令，將 hexo 套件建立在全域下。

npm install -g hexo-cli

2. 建立 hexo 專案

hexo init <資料夾名稱>

3. 進入資料夾並安裝

cd <資料夾名稱>
npm install

4. 產生靜態檔案

hexo g

5. 建立 hexo git (到_config.yml 更改配置)

npm install hexo-deployer-git --save

6. 將檔案 push 上去 (如果沒有 SSH 可能會發生問題)

hexo d

啟動 Server

hexo s

新增文章

layout -> 選擇格式
title -> 輸入標題
將新增一個 md 檔在 scaffolds 資料夾內

hexo new [layout] <title>

修改配置檔

_config.yml

#Site

title: Howie's blog // 標題
subtitle: "" // 副標題
description: "" // 描述
keywords:
author: Howie Chen // 作者
language: en
timezone: "Asia/Taipei" // 時區

# URL

## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://yoursite.com
root: /blog/  // 將此區改為github的位置

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
type: git //使用方法
repo: git@github.com:howie12207/blog.git //SSH連結
branch: master //預設分支
